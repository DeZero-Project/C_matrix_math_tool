# 簡易版matrix_math_toolテスト仕様書

## 1. テスト環境

* コンパイラ：gcc
* OS：ubuntu

## 2. テスト項目

| ID | テスト内容 | 期待される結果 | 結果（Pass/Fail） |
| --- | --- | --- | --- |
| T-01 | NULLチェック検証 | 引数にNULLを渡した場合メッセージを表示し、`exit 1`で終了すること |  |
| T-02 | 行列チェック | 行または列のサイズが異なる場合メッセージを表示し、`exit 1`で終了すること |  |
| T-03 | 演算確認結果チェック | `create_matrix`で作成した演算結果が期待どおりであること |  |
| T-04 | メモリ解放確認 | メモリリークが発生しないことを`Valgrind`で確認 | |

## 3. 異常系テスト

* **生成時のバリデーション**: `rows`または`cols`に `0` 以下の値を指定した場合に `exit 1` となること
* **メモリ確保失敗時**: `Matri`型の変数`m`がNULLだった場合、適切にエラーを表示し `exit 1` となること

## 4. 共通項目
- `exit(1)`を使う際は直前にメッセージを表示させ`$?`コマンドを使用して確認
